# Git Workflow (Стандарты Git)

## 1. Source of Truth (Источник истины)
Единственным источником истории развития проекта является репозиторий Git. История чатов с ИИ или локальные копии никогда не считаются официальной историей проекта.

## 2. Repository (Репозиторий)
Automation Engine хранится в одном Git-репозитории.

## 3. Working Copy (Рабочая копия)
Рабочей копией считается локальная папка на компьютере Mac под управлением Product Owner (Aset).

## 4. Cloud (Облачные репозитории)
* **GitHub** является основной удаленной копией репозитория (коммиты отправляются туда через `git push`).
* **Google Drive** используется исключительно как резервное холодное хранилище (backups).

## 5. Процесс по окончании каждой рабочей сессии
По завершении рабочей сессии разработчик (или ИИ-ассистент под его руководством) обязан выполнить следующие шаги:
1. Обновить всю измененную документацию.
2. Сделать Git-коммит с описанием изменений.
3. Выполнить отправку коммитов в облако (`git push`).
4. Обновить текущее состояние в [docs/development/13_PROJECT_STATE.md](file:///Users/asetmendibay/AAE%20Aset%20Automation%20Engine/docs/development/13_PROJECT_STATE.md).
5. Закрыть рабочий день в журналах.

## 6. Правило
История чатов никогда не считается источником истины проекта. Вся память и история изменений переносятся исключительно в файлы репозитория и историю коммитов Git.
