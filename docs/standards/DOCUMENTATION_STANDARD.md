# Documentation Standard (Стандарты документации)

## 1. Общие требования
* **Язык:** Документы общего планирования, правил и ценностей пишутся на русском языке. Технические спецификации API, кода и ТЗ могут содержать англоязычные термины.
* **Формат:** Markdown (GitHub Flavored Markdown).
* **Кодировка:** UTF-8.

---

## 2. Использование перекрестных ссылок (Links)
* Все ссылки на другие файлы проекта внутри документации должны использовать абсолютный путь с использованием протокола `file://` (например, `file:///Users/asetmendibay/AAE%20Aset%20Automation%20Engine/docs/core/00_PROJECT_VISION.md`).
* Ссылки на файлы **не должны** обрамляться обратными кавычками (символом `` ` ``), так как это ломает кликабельность ссылок в некоторых IDE.
  * **Неправильно:** [\`00_RULE_ZERO.md\`](file:///...)
  * **Правильно:** [00_RULE_ZERO.md](file:///...)

---

## 3. Обновление документации
* Любые изменения в архитектуре или логике работы модулей должны отражаться в каталоге документации в течение текущей рабочей сессии.
* Документ прогресса ([docs/core/04_PROGRESS.md](file:///Users/asetmendibay/AAE%20Aset%20Automation%20Engine/docs/core/04_PROGRESS.md)) и состояния ([docs/development/13_PROJECT_STATE.md](file:///Users/asetmendibay/AAE%20Aset%20Automation%20Engine/docs/development/13_PROJECT_STATE.md)) обновляются ежедневно в конце сессии.
