# Code Style Standard (Стандарты оформления кода)

## 1. Общие правила форматирования
* **Табуляция:** 2 пробела (no tabs).
* **Точки с запятой (semicolons):** Обязательны во всех TypeScript файлах.
* **Кавычки:** Одиночные (`'`) для строк в коде, двойные (`"`) только в JSON-конфигурациях.
* **Максимальная длина строки:** 100 символов (для удобства просмотра на ноутбуках).

---

## 2. Использование TypeScript
* **Строгий режим (`strict: true`):** Обязателен. Тип `any` запрещен.
* **Явное указание возвращаемых типов:** Все экспортируемые функции и методы классов должны иметь явно указанные типы возвращаемого значения.
* **Использование `null` vs `undefined`:** Использовать `undefined` для отсутствующих значений/параметров, `null` только при работе со сторонними БД или API, возвращающими null.

---

## 3. Обработка ошибок (Error Handling)
* Каждая асинхронная операция (сетевые запросы, чтение файлов, взаимодействие с Playwright) должна быть обернута в блок `try-catch`.
* Запрещено «замалчивать» ошибки. Ошибка должна быть логирована через `Logger.error()` с передачей стека вызовов.
* В критических местах (например, падение браузера) должен быть реализован механизм автовосстановления (Retry-механизм).
