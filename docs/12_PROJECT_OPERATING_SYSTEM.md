# Project Operating System (Система управления проектом)

## 1. Назначение
Automation Engine является долгосрочным инженерным проектом.  
Данный документ описывает единые правила разработки проекта, хранения документов, истории, рабочих материалов и взаимодействия участников.  
Все ИИ (искусственные интеллекты), работающие над проектом, обязаны строго соблюдать положения данного документа.

## 2. Структура проекта
Вся работа и файлы проекта организованы в следующую древовидную структуру:

```
Automation Engine/
├── docs/
├── workspace/
├── archive/
├── src/
├── tests/
├── scripts/
├── profiles/
├── logs/
└── temp/
```

## 3. Назначение папок
* **`docs/`** — Постоянная техническая и организационная документация проекта.
* **`workspace/`** — Рабочая зона для текущих задач и ежедневных материалов.
* **`archive/`** — Архив завершенных задач и материалов прошлых дней/недель.
* **`src/`** — Исходный код разрабатываемого приложения Automation Engine.
* **`tests/`** — Автоматические тесты (Unit, Integration, E2E).
* **`scripts/`** — Вспомогательные скрипты для автоматизации задач разработки, сборки или деплоя.
* **`profiles/`** — Данные профилей браузеров (куки, локальное хранилище, настройки).
* **`logs/`** — Лог-файлы работы движка и сервисов.
* **`temp/`** — Временные данные, кэш и промежуточные файлы.

## 4. Организация workspace
Текущая работа организуется по датам внутри папки `workspace/` по следующей иерархии:

```
workspace/
└── [Год]/
    └── [Месяц]/
        └── Week_[Номер недели]/
            └── [ГГГГ-ММ-ДД]/
                ├── notes.md        # Заметки
                ├── tasks.md        # Задачи дня
                ├── ideas.md        # Идеи
                ├── ai_prompts.md   # Промпты для ИИ
                └── results.md      # Результаты работы
```

## 5. Правила ежедневной работы
Каждый рабочий день имеет собственную отдельную папку формата `workspace/[Год]/[Месяц]/Week_[Номер]/[ГГГГ-ММ-ДД]/`.  
Каждая ежедневная папка должна содержать следующие файлы:
* **`tasks.md`** — Список задач на сегодня.
* **`notes.md`** — Заметки в процессе выполнения задач.
* **`ideas.md`** — Идеи, возникшие в течение дня.
* **`ai_prompts.md`** — Промпты, использованные для работы с ИИ.
* **`results.md`** — Отчет о результатах дня.

## 6. Правила завершения дня
После окончания рабочего дня выполняются следующие действия:
1. Обновляется статус задач в файле прогресса (`04_PROGRESS.md`).
2. Заполняется журнал ежедневных результатов.
3. При необходимости обновляется журнал архитектурных решений (`99_DECISIONS.md`).

## 7. Роли ИИ
В проекте определены следующие роли для участников команды (человека и ИИ):

* **Product Owner** (Владелец продукта) — **Aset**
  * Отвечает за стратегическое направление проекта;
  * Принимает финальные решения по фичам и архитектуре.

* **ChatGPT** — **Главный архитектор**
  * Отвечает за общую архитектуру и проектирование системы;
  * Разрабатывает инженерные решения и пишет код;
  * Проводит ревью архитектурных изменений;
  * Планирует задачи и составляет технические задания (ТЗ) для других ИИ;
  * Осуществляет контроль качества проекта и обновляет архитектурную документацию.

* **Claude** — **Senior Engineer (Старший инженер)**
  * Отвечает за рецензирование кода (code review);
  * Занимается поиском ошибок и уязвимостей;
  * Оптимизирует производительность и предлагает альтернативные реализации.

* **Gemini** — **Research Engineer (Инженер-исследователь)**
  * Отвечает за технологические исследования;
  * Изучает новые библиотеки, инструменты и фреймворки;
  * Находит лучшие практики и сравнивает технические решения;
  * Пишет техническую документацию по библиотекам.

* **Antigravity** (Вы) — **Project Manager (Менеджер проекта)**
  * Отвечает за создание документов и шаблонов;
  * Поддерживает структуру проекта и актуализирует файлы в `docs/`;
  * Ведет журналы прогресса и логов разработки;
  * Организует структуру папки `workspace/`;
  * Оформляет техническую документацию;
  * Переносит завершенные рабочие материалы дня/задач в архив (`archive/`).

## 8. Правила взаимодействия ИИ
* **ChatGPT** является координатором инженерной команды ИИ.
* Если во время проектирования у **ChatGPT** возникает необходимость:
  * Создать новый документ;
  * Изменить существующую документацию;
  * Изменить структуру каталогов проекта;
  * Изменить процесс разработки;
  * Перенести завершенные файлы в архив;
* **ChatGPT** формулирует полноценное техническое задание (ТЗ) для **Antigravity**.
* **Antigravity** выполняет исключительно документальную и организационную часть работы по этому ТЗ.
* Архитектурные и продуктовые решения принимает только **ChatGPT** совместно с **Product Owner (Aset)**.

## 9. Правила изменений
Ни один ИИ не имеет права изменять проектную документацию (файлы в `docs/`) самостоятельно по своей инициативе. Все изменения проектной документации должны строго проходить через техническое задание (ТЗ) от координатора проекта (ChatGPT) или по прямому указанию Product Owner.

## 10. Жизненный цикл задачи
Любая задача в рамках проекта строго проходит следующий цикл:

```
Идея ➔ Обсуждение ➔ Архитектура ➔ План ➔ Реализация ➔ Проверка ➔ Документация ➔ Ретроспектива ➔ Следующая задача
```

## 11. Правила Git
После каждого успешно завершенного этапа задачи обязательно выполняются:
* Создание Git-коммита с понятным описанием изменений.
* Обновление документации.
* Актуализация файла прогресса (`04_PROGRESS.md`).

## 12. Цель
Архитектура и организация проекта построены так, чтобы через несколько лет любой новый инженер или ИИ-ассистент, впервые открыв проект, мог полностью понять его устройство и правила работы менее чем за один час.

## 13. Project Memory (Память проекта)
* Источником памяти проекта являются документы.
* Ни один ИИ не считается носителем знаний проекта.
* Любой новый ИИ обязан восстановить контекст исключительно по документации.
