# Handoff Protocol (Протокол передачи задач)

Этот документ описывает регламент передачи задач между различными ИИ-участниками команды во избежание потери контекста и архитектурного дрейфа.

## 1. Схема передачи задачи (Handoff Flow)

Каждая задача на разработку кода проходит следующую цепочку передачи ответственности:

```
[ChatGPT: Архитектор] ➔ Проектирует задачу и формирует ТЗ
         ↓
[Claude: Senior] ➔ Делает архитектурный аудит и ревью ТЗ
         ↓
[ChatGPT: Архитектор] ➔ Вносит правки (при наличии) и назначает Codex
         ↓
[Codex: Developer] ➔ Пишет код и тесты
         ↓
[Claude: Senior] ➔ Проводит Code Review
         ↓
[ChatGPT: Архитектор] ➔ Утверждает Pull Request и ставит задачу Antigravity
         ↓
[Antigravity: PM] ➔ Фиксирует изменения в доках и закрывает задачу
```

---

## 2. Требования к Handoff-сообщению
При передаче задачи следующему участнику (ИИ), передающий обязан составить Handoff-отчет, содержащий:
1. **ID задачи:** Ссылка на файл задачи (например, `TASK_AE_00X`).
2. **Что сделано:** Перечень созданных файлов, функций и тестов.
3. **Текущий статус проверки:** Результаты выполнения тестов.
4. **Инструкция для следующего шага:** Что именно должен сделать принимающий ИИ.

---

## 3. Правило фиксации контекста
Ни один ИИ не должен передавать задачу на следующий этап, если тесты падают, либо если изменения затрагивают интерфейсы других модулей без предварительного согласования с **ChatGPT** (Chief Architect).
