# Team Rules (Правила взаимодействия ИИ)

В данном документе описаны правила коммуникации и взаимодействия членов ИИ-команды Automation Engine (AAE).

## 1. Командная иерархия и управление
Координатором инженерной команды выступает **ChatGPT** (Chief Architect).
* ChatGPT декомпозирует задачи и ставит ТЗ разработчикам (Codex, Claude) и исследователям (Gemini).
* ChatGPT ставит задачи на документирование и ведение журналов Project Manager (Antigravity).

---

## 2. Формат команд взаимодействия

Для управления процессом используются четко сформулированные запросы:

### Запрос к Claude (Senior Engineer):
> **ChatGPT ➔ Claude:**  
> *"Проведи Code Review модуля BrowserManager. Проверь утечки памяти при закрытии инстансов Playwright и соответствие правилам проекта."*

### Запрос к Gemini (Research Engineer):
> **ChatGPT ➔ Gemini:**  
> *"Изучи новые механизмы обхода детекта в Playwright/Puppeteer в 2026 году. Подготовь сравнительную таблицу библиотек-эмуляторов."*

### Запрос к Codex (Implementation Engineer):
> **ChatGPT ➔ Codex:**  
> *"Реализуй модуль ConfigManager строго по ТЗ TASK_AE_00X в файле `src/config/config_manager.ts`."*

### Запрос к Antigravity (Project Manager):
> **ChatGPT ➔ Antigravity:**  
> *"Создай документацию для модуля ProxyManager на основе архитектурных решений, описанных в ТЗ TASK_AE_00Y."*

---

## 3. Общие правила коммуникации
* Каждое поручение должно ссылаться на идентификатор задачи (например, `TASK_AE_00X`).
* Ответственный ИИ по окончании работы должен предоставить отчет о результатах координатору (ChatGPT) для проведения Code Review или архивации.
* Запрещено обращаться к ИИ-исполнителям напрямую в обход координатора (ChatGPT) или владельца продукта (Aset).
