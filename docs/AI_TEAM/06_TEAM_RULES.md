# Team Interaction Rules

## 1. Источник задач

Только Product Owner принимает Idea. Repository Chief Engineer преобразует её в
task specification только после проверки источника в Project Roadmap 0.1.

## 2. Независимость ролей

- Advisor, Review и Research не подменяют друг друга.
- Ни одна внешняя роль не утверждает merge или архитектурное решение.
- Codex предоставляет внешним ролям факты, diff, результаты проверок и точный
  вопрос; не просит их выполнять неограниченную работу за пределами задачи.

## 3. Формат результата

Каждый результат содержит task/ADR ID, scope, evidence, findings, ограничения и
рекомендованный следующий шаг. Предположения маркируются как предположения.

## 4. Источник процесса

Единственный порядок взаимодействия определён в
`docs/core/11_WORKFLOW.md`. Этот документ не устанавливает альтернативный
workflow.
