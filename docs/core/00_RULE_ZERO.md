# Rule Zero

Любое изменение в проекте должно сделать Automation Engine лучше.

Любое решение должно уменьшать будущую сложность проекта, а не только решать текущую задачу.

Если изменение:

• усложняет архитектуру;

• ухудшает читаемость;

• увеличивает связность;

• ухудшает масштабируемость;

• ломает модульность;

значит сначала нужно найти другое решение.

Проект важнее отдельной функции.

Архитектура важнее скорости.

Долгосрочная поддержка важнее краткосрочной выгоды.
