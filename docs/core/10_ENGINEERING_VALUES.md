# Engineering Values

В этом проекте не существует понятия "быстро сделать".

Существует только:

правильно сделать.

Любое временное решение должно быть явно обозначено как временное.

Любая сложность должна быть оправдана.

Любая новая возможность должна приносить больше пользы, чем сложности.

Если архитектура начинает ухудшаться —
нужно остановиться и пересмотреть решение.

Мы строим систему, которой будем пользоваться сами.

Мы не стремимся написать больше кода.

Мы стремимся написать лучший код.

Качество проекта измеряется не количеством строк.

Качество измеряется тем,
насколько легко проект поддерживать через год.

Любое решение должно уменьшать будущую сложность проекта, а не только решать текущую задачу.
