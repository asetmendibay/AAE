# Architecture Freeze

**Статус:** действует с Project Baseline 0.1
**Основание:** Решение №2 в `docs/core/99_DECISIONS.md`.

## Зафиксированные области

Без ADR или явно зафиксированного решения Product Owner запрещено:

- менять верхнеуровневую структуру каталогов проекта;
- менять границы слоёв и направление зависимостей;
- менять единственный development workflow;
- добавлять, удалять или переопределять постоянные роли AI Team;
- менять constitution любой постоянной роли;
- массово переписывать или перемещать документацию;
- выполнять реорганизацию без измеримой пользы, риска и плана проверки.

## Допустимые изменения без ADR

- Реализация задач, прямо вытекающих из active roadmap stage.
- Добавление module-specific файлов внутри уже утверждённых границ.
- Исправление ошибок, ссылок, опечаток и несоответствий документации фактам.
- Уточнение task specifications без изменения baseline.

## Объективная необходимость

Предложение об изменении должно содержать: наблюдаемый предел baseline,
затронутые элементы, альтернативы, измеримую пользу, риски миграции и план
проверки. Отсутствие удобства реализации не является достаточной причиной.

## Порядок исключения

1. Repository Chief Engineer фиксирует необходимость и готовит ADR.
2. Independent Architecture Advisor при необходимости даёт независимую оценку.
3. Product Owner принимает или отклоняет решение.
4. Только после фиксации решения изменение может стать задачей workflow.
