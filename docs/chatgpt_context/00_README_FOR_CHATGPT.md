# Восстановление контекста для ChatGPT

**Назначение:** Главная инструкция по восстановлению контекста для роли `Independent Architecture Advisor` (ChatGPT).

## 1. Цель папки `docs/chatgpt_context/`

Эта папка содержит минимальный, стабильный и долгосрочный слой документации, предназначенный исключительно для быстрого восстановления архитектурного контекста ChatGPT.
ChatGPT больше не требуется знать каждую техническую деталь проекта и исходного кода — полным контекстом репозитория владеет `Repository Chief Engineer` (Codex). ChatGPT необходим только верхнеуровневый архитектурный контекст для эффективного ведения архитектурных сессий.

## 2. Порядок чтения документов

При начале нового архитектурного диалога ChatGPT должен прочитать документы строго в следующем порядке:

1. [00_README_FOR_CHATGPT.md](00_README_FOR_CHATGPT.md) (этот документ) — Назначение папки и правила.
2. [06_AI_BRIEF.md](06_AI_BRIEF.md) — Оперативный срез (последний статус, риски, решения).
3. [01_EXECUTIVE_CONTEXT.md](01_EXECUTIVE_CONTEXT.md) — Краткое описание проекта, текущая версия, этап и цель.
4. [02_CURRENT_STATE.md](02_CURRENT_STATE.md) — Актуальное фактическое состояние без истории.
5. [03_ACTIVE_DECISIONS.md](03_ACTIVE_DECISIONS.md) — Действующие архитектурные решения.
6. [04_NEXT_TARGET.md](04_NEXT_TARGET.md) — Описание текущей вехи, ожидаемого результата и следующего этапа.
7. [05_ARCHITECTURE_SUMMARY.md](05_ARCHITECTURE_SUMMARY.md) — Структура модулей, границы ответственности и ключевые принципы.
8. [07_SESSION_HANDOFF.md](07_SESSION_HANDOFF.md) — Состояние передачи контекста между сессиями (шаблон и записи).

После прочтения этих 8 документов в указанном порядке архитектурный контекст считается полностью восстановленным.

## 3. Источник истины (Source of Truth)

В проекте действует строгое правило соподчинения источников истины (от абсолютного к производному):

```
Git Repository
↓
Documentation
↓
Executive Context
↓
ChatGPT Context Layer
↓
AI Sessions
```

- **История чатов никогда не считается источником истины.**
- Документы в этой папке представляют собой выжимку и всегда должны быть синхронизированы со следующими каноническими источниками:
  - **Executive Context:** [00_EXECUTIVE_CONTEXT.md](../executive/00_EXECUTIVE_CONTEXT.md)
  - **Core Documentation & Architecture:** `docs/core/` (в частности, [01_MASTER_PLAN.md](../core/01_MASTER_PLAN.md), [02_ARCHITECTURE.md](../core/02_ARCHITECTURE.md), [12_ARCHITECTURE_FREEZE.md](../core/12_ARCHITECTURE_FREEZE.md))
  - **Decisions:** [99_DECISIONS.md](../core/99_DECISIONS.md)
  - **AI Team:** [00_TEAM.md](../AI_TEAM/00_TEAM.md)

## 4. Исключённые документы (Что можно не загружать)

Для работы ChatGPT в качестве Independent Architecture Advisor **не требуется** загружать:

- Исходный код проекта (`src/`) и тесты (`tests/`).
- Детальные стандарты кодирования и Git (`docs/standards/`).
- Шаблоны (`docs/templates/`).
- Исторические журналы изменений и архивы (`archive/`, `logs/`, `temp/`).

Это значительно уменьшает объём синхронизации и предотвращает переполнение контекста ChatGPT.

## 5. Правила обновления папки

- **Минимизация размера:** Размер ChatGPT Context Layer должен оставаться минимальным. Если документ начинает разрастаться и дублировать основную документацию, его необходимо сократить до архитектурной выжимки со ссылками на первоисточники. Цель этой папки — быстро восстановить контекст, а не заменить всю документацию проекта.
- Обновление выполняется только после завершения значимого этапа разработки.
- **Запрещено** обновлять документы после каждого коммита.
- Документы обновляются только в следующих случаях:
  1. Завершение этапа Roadmap.
  2. Принятие нового архитектурного решения (ADR/Decision).
  3. Изменение Roadmap (Master Plan).
  4. Изменение текущей цели проекта.
  5. Изменение состава AI Team или ролей участников.
