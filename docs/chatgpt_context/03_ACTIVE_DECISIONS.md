# Active Decisions

**Назначение:** Реестр действующих архитектурных решений, обязательных к исполнению. Без истории обсуждений.

## 1. Действующие решения

- **D-001 (Стек автоматизации):** Отказ от Browser Automation Studio в пользу собственного модульного движка на Node.js 20+, TypeScript (строгая типизация) и адаптера Playwright.
- **D-002 (Архитектура модулей):** Архитектура строится как локальный модульный монолит. Зависимости направлены внутрь к ядру (`core`). Запрещен прямой импорт Playwright в сценариях (модулях) — управление браузером изолировано портами и адаптерами.
- **D-003 (Изменения и Baseline):** Любые изменения baseline, структуры каталогов, постоянных ролей ИИ и workflow требуют создания ADR или явного согласования с Product Owner. Отсутствие явного решения в baseline не является разрешением на его самостоятельный выбор.
- **D-004 (Жизненный цикл памяти):** Все результаты, решения и история проекта хранятся в репозитории Git и проектной документации. Вне этих источников «память» проекта отсутствует.
- **D-005 (Порядок работ):** Задачи выполняются строго последовательно согласно Master Plan. Новые задачи формируются только путём декомпозиции этапов плана.

## 2. Предстоящие решения (Stage-gated decisions)

Эти решения должны быть приняты до начала соответствующих этапов:

- **Перед Этапом 2:** Формат контрактов Config/Logger и таксономия ошибок.
- **Перед Этапом 3:** Статусы Job и классификация retry-попыток.
- **Перед Этапом 4:** Формат диагностических артефактов.
- **Перед Этапом 5:** Спецификация хранилища профилей и настройки сети.
- **Перед Этапом 6:** Конкретный целевой ресурс для первого модуля автоматизации.
- **Перед Этапом 8:** Механизм дистрибуции релизов.
